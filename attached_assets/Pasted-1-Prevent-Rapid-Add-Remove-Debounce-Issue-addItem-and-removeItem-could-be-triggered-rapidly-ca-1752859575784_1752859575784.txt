1. Prevent Rapid Add/Remove (Debounce)

Issue: addItem and removeItem could be triggered rapidly, causing race conditions or API spam.

Fix:
Add a simple debounce function for add/remove:

js
Copy
Edit
// At the top of the file
const debounce = (fn, delay = 300) => {
  let timeout;
  return (...args) => {
    clearTimeout(timeout);
    timeout = setTimeout(() => fn(...args), delay);
  };
};

// When using addItem/removeItem:
const debouncedAddItem = debounce(addItem);
const debouncedRemoveItem = debounce(removeItem);

// Then replace usage in the JSX:
// onClick={() => debouncedAddItem(...)}
// onClick={() => debouncedRemoveItem(...)}
